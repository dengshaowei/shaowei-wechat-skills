# Shaowei Wechat Article

## Role

这个 skill 是少威公众号体系里的**内容生成层**。

它负责：
- 把原始素材整理成完整文章
- 补齐逻辑、结构、转场和落点
- 保持少威固定文风
- 输出适合后续排版的文章中间稿

它**不负责**：
- 最终固定渲染
- dry-run 检查
- 草稿箱发布

## Position In The Skill Suite

默认串联关系：

1. `shaowei-wechat-article`：内容生成
2. `shaowei-wechat-writer`：格式渲染与定稿
3. `wechat-article-publisher`：预览与发布
4. `shaowei-wechat-pipeline`：总控编排

## Inputs

可接受的输入类型：
- 原始初稿
- 问答稿 / 采访稿
- 提纲
- 碎片观点
- 网页链接提炼后的内容
- 会议纪要 / 聊天记录 / 随手笔记

## Output Contract

输出目标是**可继续加工的公众号正文稿**，默认满足：

- 有明确标题
- 有完整开头、中段、结尾
- 内容逻辑完整，适合公众号阅读
- 段落尽量偏短，但不强制写死最终 HTML 样式
- 结尾已经回到价值判断、成长、行动、选择或现实理解
- 若用户没有特别指定，可附一个互动问题或开放讨论口

建议输出形态：
- Markdown 文章稿
- 或者便于下游继续渲染的纯正文结构稿

## Writing Rules

### 1. 开头
默认从以下几类切口进入：
- 生活感
- 现实感
- 反差感
- 常见误解
- 一个容易共鸣的提问

避免一上来就定义概念。

### 2. 中段结构
默认优先使用少威固定结构：
- 生活切口
- 技术拆解
- 抽象升维
- 落到用户价值 / 职业判断 / 成长思考

### 3. 语言要求
- 理性但有温度
- 不要学术腔
- 不要空洞鸡汤
- 尽量把技术问题翻译成人能直接理解的话
- 避免为了显得专业而故意写复杂

### 4. 结尾
结尾不要突然收住。

默认要做到：
- 回到更高一层判断
- 给出一个可记住的落点
- 必要时留一个互动问题

## Non-Negotiables

- 不要把原始素材原封不动拼接成文章
- 不要写成答题卡式说明文
- 不要上来就堆概念定义
- 不要把文章写成大段、重解释、低可读性的结构
- 不要直接替代 `shaowei-wechat-writer` 的固定渲染职责

## Portability

这个 skill 默认设计成**工具无关**：
- 在 OpenClaw 里可直接与后续 skill 串联
- 在其他 agent / 软件里，也可把它当成“公众号内容生成器”单独复用
- 只要后续系统接受 Markdown 或正文结构稿，都可以接这个 skill 的输出
