# Shaowei Wechat Writer

## Role In The Skill Suite

这个 skill 是少威公众号体系里的**格式渲染层 / 定稿层**。

它主要负责：
- 接收原始稿件或中间稿
- 按少威固定公众号样式进行排版和视觉结构整理
- 补齐重点句、标题层级、留白、尾注和收尾结构
- 输出适合直接预览或发布的最终文章稿

它不负责：
- 微信接口发布
- 草稿箱提交
- 发布状态查询

默认串联关系：
- 上游可接 `shaowei-wechat-article`
- 下游默认接 `wechat-article-publisher`
- 若用户要求一条龙处理，则由 `shaowei-wechat-pipeline` 编排调用

## Canonical Template Source

## Required Layout Rules

### 1. 标题后的编号不是普通文本

标题下方不是简单一行 `No.xx`，而是一个**浅黄底、居中、圆角边框的提示框编号块**。

目标效果特征：

- 居中
- 浅黄底
- 有边框
- 有圆角
- 视觉上像提示卡片，而不是正文段落

### 2. 正文主体格式

- 保持短段落
- 段间留白清晰
- 默认一句一段 / 一层意思一段
- 转折句和结论句优先单独成段
- 正文默认左对齐/两端分布式阅读体验
- 不要擅自把正文改成另一种完全不同的版式
- 不要写成一大段密集文字

### 3. 分节标题格式

分节标题应尽量模拟样文效果：

- 浅红底
- 左侧竖条强调
- 加粗
- 明显区别于正文段落

### 4. 重点句格式

可对关键句使用样文中的强调方式：

- 浅黄底高亮
- 深色加粗文字

补充要求：

- 正文最后的“最后想说”部分，通常必须有重点句
- 若有核心落点、最终判断、金句，应优先高亮出来

### 5. 列举内容格式

列举内容不要只用一种排法，需要按内容选择：

- **形式 A：轻列举** —— 一句一行，适合保持轻节奏
- **形式 B：卡片式列表** —— 适合一组并列能力、步骤、问题、维度

若是一组完整并列项，优先考虑卡片式列表，而不是全部拆成散段。

### 6. 图片与图注

若有图片：

- 图片居中
- 图片下方可带一行简短图注
- 图注通常居中、轻量、可带斜体感

### 7. 结尾结构

正文结束后：

- 先保留较明显空白
- 再放**居中的长分隔线**
- 再单独一行放**居中的 `End`**
- 然后再追加固定个人介绍尾注

注意：`End` 在样文中是单独收尾标记，不要混进正文最后一段。

## Writing Style Rules

在版式符合样文的前提下，再贴合长期文风：

- 生活化、带反差感的切口开头
- 善于把技术问题翻译成普通人可感知的类比
- 常用结构：生活切口 -> 技术拆解 -> 抽象升维 -> 人生/成长落点
- 语气理性但有温度，避免学术腔和空洞鸡汤
- 段落偏短，阅读压力低
- 中段和结尾可适度放入可传播金句
- 收束时常回到选择、成长、勇气、行动

## Fixed Tail

除非用户明确说不要，否则文章末尾始终追加，并按以下排版执行：

🌟 关于我：ENFJ-A | 终身成长者 | 务实的理想主义者

📝 在这里，我们可以一起探讨：
🤖 具身智能转行：我的实战经验与行业观察；
📅 个人成长规划：如何拆解目标，让理想落地；
✨ 生活感悟复盘：自我突破与相信的力量。

请相信，我们每个人都与众不同，也终将实现自己的愿望。
保持联系 🔗 让我们一起记录，一起向上生长 🌱
围观朋友圈：dengdengdengdeng4101（备注：公众号）
小红书/即刻：少威shaowei

排版要求：

- 不要压成一个整段
- 也不要拆得过碎
- 按用户最终确认的分行方式执行

## Non-Negotiables

- 不要再把编号写成普通裸文本段落，除非用户明确要求
- 不要把正文整体改成与样文完全不同的版式
- 不要忽略分节标题、重点句、留白节奏这些样文特征
- 不要把正文写成一大段密文
- 不要忽略列举内容的层级，需要按内容选择轻列举或卡片式列表
- 不要把结尾分隔线和 `End` 改成非居中
- 若用户说“按我的格式”，优先参考样文 + 已确认规范，而不是口头抽象规则
