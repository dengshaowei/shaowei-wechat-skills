## 一、总原则

后续所有公众号文章，默认同时遵守这三层：

1. **渲染格式固定**
2. **文字风格固定**

核心原则不是“写得像模板”，而是：

**写出来要像少威会发的公众号，并且读起来像公众号，而不是像规则说明文。**

## 三、渲染格式 SOP

## 1. 标题区规则

### 标题
- 标题正常显示在公众号系统标题区
- **正文中不要重复出现标题**

### 标题下编号框
标题下必须紧接一个编号框，不要写成普通裸文本 `No.xx`。

固定特征：
- 居中
- 浅黄底
- 细边框
- 圆角
- 紧跟标题出现
- 视觉上像提示卡片

参考结构：

```html
<p align="center" style="margin: 14px 0 24px 0;">
  <span style="display:inline-block;padding:8px 18px;background:#FFF7E6;border:1px solid #F3D9A4;border-radius:8px;color:#5F5130;font-size:15px;line-height:1.8;">
    No.xx
  </span>
</p>
```

规则补充：
- 若用户未指定编号，保留 `No.xx`
- 若用户指定编号（如 `No.35`），按指定值执行

---

## 2. 正文段落规则

### 基础样式
正文统一采用：
- 16px 左右字号
- 舒展行高
- 深灰字色
- 两端对齐
- 无首行缩进
- 段距统一

### 断句与公众号易读性
这是强约束。

以后正文必须满足：
- 一句话说完就隔开
- 一层意思一段
- 转折句单独成段
- 结论句单独成段
- 不写成长段密文
- 不写成论文感
- 不要“样式块很对，但读起来不像公众号”

### 最新确认原则
以后正文排布要：

**优先按“昨天校准稿”的公众号易读节奏执行，而不是只按抽象 HTML 样式拼接。**

也就是说：

**阅读呼吸感 > 机械样式感**

---

## 3. 一级标题规则

一级标题统一使用红标块。

固定特征：
- 浅红底
- 左侧红色竖条
- 深红文字
- 加粗
- 和正文有明显层级区分

参考结构：

```html
<h2 style="margin:1.45em 0 0.72em;padding:0.46em 0.72em;font-size:1.2em;line-height:1.45;color:#7a1f1f;font-weight:800;background:#fff2f2;border-left:4px solid #ef4444;border-radius:4px;">一、一级标题</h2>
```

规则：
- 一级标题统一红标块
- 二级拆分可以更轻，不要所有标题都同样重

---

## 4. 重点句规则

重点句统一采用：
- 浅黄底高亮
- 深色加粗

适用场景：
- 核心判断
- 关键转折
- 金句
- 段落结论句
- 文末“最后想说”里的关键落点

参考结构：

```html
<strong style="font-weight:800;color:#9a3412;background:#fff3d6;padding:0 2px;border-radius:3px;">重点句</strong>
```

规则：
- 不要整段全高亮
- 不要一节里密集高亮
- 高亮的目的是“提重点”，不是装饰满屏

### 强约束
- **“最后想说”这一部分一般必须有重点句**
- 若有最终落点、核心判断、收束句，要优先高亮

---

## 5. 列举内容规则

列举不能永远只用一种形式。

### A. 轻列举
适合：
- 节奏轻
- 内容短
- 想保留呼吸感
- 更像公众号下滑阅读

示例：
- 写文案。
- 做摘要。
- 查资料。
- 辅助编程。

### B. 卡片式列表
适合：
- 一组完整并列项
- 需要快速扫点
- 属于步骤项 / 能力项 / 问题项 / 维度项

规则：
- 并列关系强时，优先卡片式列表
- 节奏更重要时，优先轻列举
- 不要整篇只剩一种列举方式

---

## 6. 图片与图注

如正文需要插图，可采用：
- 图片居中
- 图片下方添加轻量图注
- 图注不喧宾夺主

目前这部分为允许项，按文章需要使用。

---

## 7. 结尾区规则

### 分隔线
- 正文结束后出现一条长分隔线
- **必须居中**

### End
- 单独一行
- **必须居中**

这是已经确认过的最终规则。

参考结构：

```html
<p style="margin:36px 0 8px 0;text-align:center;letter-spacing:0.08em;color:#6b7280;">────────────</p>
<p style="margin:0 0 28px 0;text-align:center;color:#6b7280;">End</p>
```

---

## 8. 固定尾注规则

除非用户明确说不要，否则默认保留固定尾注。

尾注内容包括：
- 关于我
- 可探讨主题
- 联系方式
- 收束文案

排版要求：
- 不压成一整坨
- 也不要拆得过碎
- 必须按最终确认版分行方式执行

## 四、文字 / 文风 SOP

## 1. 开头方式
默认不要一上来就讲抽象概念。

推荐开头方式：
- 从生活感切入
- 从现实感切入
- 从反差感切入
- 先把读者带进场景，再进入概念

---

## 2. 中段结构
默认结构为：

**生活切口 → 技术拆解 → 抽象升维 → 人生 / 成长 / 选择落点**

---

## 3. 表达气质
整体语气要求：
- 理性
- 有温度
- 不学术腔
- 不空洞鸡汤
- 善于把复杂问题翻译成普通人能理解的话

原则：

**不是卖弄理解，而是帮助理解。**

---

## 4. 内容推进方式
推进时默认遵守：
- 多短段落
- 少长大段
- 中间可有可传播金句
- 重要转折单独拉出
- 结尾常回到选择、成长、勇气、行动、现实理解

---

## 5. 收尾方式
收尾一般不是硬总结，而是：
- 提升一层理解
- 给出最终判断
- 落到人生 / 成长 / 选择 / 行动
- 必要时留一个互动讨论口

---

## 五、完整工作流 SOP

### Step 1：接收输入
用户可以提供：
- 初稿
- 提纲
- 零散想法
- 待改写文章
- 网页链接
- 录音转文字
- 主题 + 核心观点

输入不要求完整，也不要求工整。

### Step 2：判断素材类型
识别输入属于：
- 完整初稿
- 提纲型内容
- 碎片型内容
- 链接型内容
- 半成品改写任务

然后补全结构、逻辑与表达。

### Step 3：按少威文风写成公众号文章
这一阶段负责：
- 结构重组
- 逻辑增强
- 语言润色
- 风格对齐
- 必要时扩写 / 收束 / 重写

目标不是“改病句”，而是：

**把它变成像少威会发出来的文章。**

### Step 4：按渲染格式排版
这一阶段负责：
- 去掉正文重复标题
- 补编号框
- 按公众号易读节奏断句
- 一级标题上红标块
- 重点句高亮
- 按需要选择轻列举 / 卡片式列表
- 结尾加分隔线 + End
- 接固定尾注

### Step 5：发布前检查
发布前至少检查：

#### 内容检查
- 开头是否有吸引力
- 中段逻辑是否顺畅
- 有没有过度说教
- 结尾是否有落点
- 是否保留少威个人表达感

#### 渲染检查
- 标题是否没在正文重复
- 标题下是否有编号框
- 正文是否短段落、可下滑阅读
- 一级标题是否红标块
- 重点句是否有但不过密
- 列举方式是否选得合理
- 分隔线是否居中
- End 是否居中
- 尾注是否按固定排法保留

### Step 6：推送到公众号草稿箱
默认流程：

**成稿 → 微信公众号草稿箱**

默认行为：
- 创建草稿
- 不直接正式发布

也就是说：

**默认是“先发草稿箱，最后由用户自己确认”。**
