# 少威公众号图文 SOP（精细版）

## 1. 适用范围

适用于以下场景：
- 用户给初稿，要求直接生成公众号成品
- 用户要求配图、生图、回填正文并进草稿箱
- 用户要求解释型文章使用中文白板教授图

## 2. 总流程

1. 内容整理
2. 固定格式渲染
3. 判断是否启用配图闭环
4. 配图规划
5. 生图执行
6. 上传微信素材
7. 回填正文图片 URL
8. 设置图注样式
9. 重发草稿箱
10. 检查最终渲染

## 3. 内容与格式规则

- 标题使用 frontmatter `title:`，不要把标题再写成正文一级标题
- 标题下使用 `No.xx` 浅黄编号框
- 正文短段落
- 一级标题红标块
- 关键句浅黄高亮
- 结尾使用分隔线 + `End`
- 固定尾注按确认版分行

## 4. 配图策略

### 解释型 / 方法型 / 拆解型文章
优先使用三段式：
1. 开头总览图
2. 中段关系解释图
3. 结尾映射 / 升维图

默认优先使用：
- 中文白板教授图
- 带文字关系图
- 讲解图

### 情绪型 / 故事型文章
优先使用：
- 场景图
- 情绪图

## 5. 中文白板教授图规则

必须满足：
- 有可读中文文字
- 有箭头、框、连接关系
- 白底或浅底
- 像老师讲课板书图
- 图承担解释功能

不要：
- 抽象装饰图
- 无字概念图
- 只有氛围感的图

## 6. 生图执行规则

- 每张图生成单独 request json
- 调 OpenAI-compatible images 接口
- 保存 response json
- 解码 base64 成 png
- 命名按 cover / concept / ending 区分

## 7. 微信正文图片规则

- 本地图片不能直接进正文
- 必须先上传微信素材库
- 正文图片 src 使用微信素材 URL

## 8. 图注规则

默认：
- 不在正文里解释图片
- 只在图下放简短图注
- 格式优先：`图 1｜xxx`

可选视觉加强：
- 居中图注
- 胶囊图注
- 往图片内部压一点
- 按微信渲染观感做右移补偿

## 9. 发布规则

- 默认只发草稿箱
- 默认用 `python3`
- 发布后记录 `draft_media_id`

## 10. 最终检查

检查：
- 标题是否重复
- 格式是否符合固定模板
- 图片是否显示正常
- 图片位置是否正确
- 图注样式是否符合当前确认版本
- 草稿箱是否发布成功
