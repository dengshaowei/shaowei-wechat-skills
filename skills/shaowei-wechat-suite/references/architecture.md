# 少威公众号 skill 套件结构说明

## 当前默认闭环（无生图 / 无自动配图）

### 1. 内容生成层
- skill: `shaowei-wechat-article`
- 输入：初稿 / 提纲 / 问答 / 碎片观点 / 链接内容
- 输出：可继续加工的公众号文章中间稿

### 2. 固定格式渲染层
- skill: `shaowei-wechat-writer`
- 输入：文章中间稿或待整理正文
- 输出：符合少威固定公众号样式的定稿

### 3. 发布层
- skill: `wechat-article-publisher`
- 输入：Markdown / 已排版稿件
- 输出：dry-run 预览、draft_media_id、可选 publish_id

### 4. 总控编排层
- skill: `shaowei-wechat-pipeline`
- 负责按顺序串联上述各层

## 当前推荐串联顺序

1. 先写内容
2. 再固定格式渲染
3. 再预览
4. 最后推草稿箱

## 可选扩展层（暂不进入默认闭环）

### 配图规划层
- skill: `shaowei-wechat-illustrator`
- 输入：定稿后的文章内容
- 输出：图片插入建议、图片说明、图文节奏规划、可选封面建议

当前先作为扩展能力保留。
等用户后续把配图 skill 补完整后，再正式插入主流程，位置在“固定格式渲染”和“预览”之间。

## 为什么现在先不把配图并入默认闭环

因为用户当前明确要求先完成“无生图版本”的可用闭环。

这样做的好处：
- 先让写作、排版、发布主链路稳定可复用
- 不让未完成的配图能力影响主流程
- 等配图 skill 成熟后再无缝插入
